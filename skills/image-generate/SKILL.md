# Image Generate

## 适用场景
当需要根据文本描述生成图片时，使用该技能调用 `image_generate` 函数。

## 功能特性
- ✅ 支持飞书/企业微信等渠道直接发送图片预览，用户无需手动下载
- ✅ 自动处理 API 认证和权限获取
- ✅ 自动下载图片到本地目录

## 使用步骤
1. 准备清晰具体的 `prompt`。
2. 运行脚本 `python scripts/image_generate.py "<prompt>"`。运行之前cd到对应的目录。
3. 脚本将下载生成的图片到本地目录（默认当前目录，可通过 `IMAGE_DOWNLOAD_DIR` 环境变量指定）。
4. 使用 `message` 工具将生成的图片文件直接发送给用户预览。
5. 发送图片文件时必须使用**绝对路径**（例如 `/root/.openclaw/...`），禁止使用相对路径或 `~` 开头的路径，否则会导致上传失败。

## 认证与凭据来源
- 优先读取 `MODEL_IMAGE_API_KEY` 或 `ARK_API_KEY` 环境变量。
- 若未配置，将尝试使用 `VOLCENGINE_ACCESS_KEY` 与 `VOLCENGINE_SECRET_KEY` 获取 Ark API Key。

## 成功标准
一次图片生成任务，只有满足以下条件之一，才算真正完成：
1. 已在当前聊天会话中发送图片预览；
2. 已向用户返回可访问的图片 URL；
3. 若以上都无法完成，已明确说明失败原因。
**仅生成出本地文件路径，不算任务完成。**

## 输出格式
- 图片文件将被下载到指定本地路径。
- 若调用失败，将打印错误信息。

## 飞书发送规则（强制）
当 `channel=feishu` 时，必须遵守以下规则：
1. 如果用户要求预览图片，或生成结果是图片内容，应优先在飞书会话中发送图片预览。
2. 必须优先使用机器人应用身份发送消息，即：
   - `im:message:send_as_bot`
3. 如果用户已经明确指定使用 `im:message:send_as_bot`，必须遵守，不得忽略。
4. 发送图片时必须使用**绝对路径**（例如 `/root/.openclaw/...`），禁止使用相对路径或 `~` 开头的路径，否则会导致上传失败。
5. 不得仅返回本地文件路径替代图片发送。
6. 不得要求用户自行访问云端路径或工作区路径。
7. 若发送成功，可简要回复"图片已发送，请查看当前聊天窗口中的预览"。

## 禁止行为
- 不要把本地文件路径当作用户可预览结果。
- 不要在云端运行场景下让用户访问 `~/.openclaw/...` 路径。
- 不要在用户明确要求发送图片时，仅返回路径或仅说明已生成。
- 不要忽略"以 `im:message:send_as_bot` 身份发送"的用户要求。
- 不要将"生成完成"误判为"任务完成"；生成后仍需完成交付。

## 示例
```bash
# 生成图片
cd ~/.openclaw/workspace/skills/image-generate
python scripts/image_generate.py "一只可爱的猫"
```
